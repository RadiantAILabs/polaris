# @polaris/types

Generated TypeScript bindings for Polaris wire types. The source of truth is the Rust workspace at the repo root — every file in `src/` (except `index.ts`) is regenerated by [`ts-rs`](https://github.com/Aleph-Alpha/ts-rs) from a Rust type annotated with `#[derive(TS)]`.

## Layout

```
bindings/
  ts/                      ← this package (@polaris/types)
    package.json
    tsconfig.json
    src/
      index.ts             ← hand-maintained barrel
      Manifest.ts          ← generated
      Section.ts           ← generated
      ...
  python/                  ← reserved for future Python bindings
  ...
```

`bindings/` is the cross-language root; `bindings/ts/` is one of potentially many language targets. Other TypeScript packages (e.g. a future SDK or dashboard plugin scaffold) can live alongside as sibling directories under `bindings/ts/`.

## Consuming

While unpublished, depend on the package via a workspace path or a Git submodule:

```jsonc
// dashboard repo's package.json
{
  "dependencies": {
    "@polaris/types": "file:../polar-rs/bindings/ts"
  }
}
```

```ts
import type { Manifest, Panel, SessionMetadata } from "@polaris/types";
```

Modern bundlers (Vite, esbuild, swc) consume `.ts` directly via the `"types"` and `"exports"` fields. No build step is required at the consumer side.

## Regenerating

From the workspace root:

```bash
cargo test --features typegen
git add bindings/ts/src
```

CI runs `git diff --exit-code bindings/`, so any uncommitted regeneration drift fails the build.

## Adding a new type to the barrel

`src/index.ts` is hand-maintained. After adding a `#[derive(TS)]` to a Rust type, regenerate (above), then add the corresponding `export type` line to `src/index.ts`. Keep entries alphabetized.

See [`docs/reference/typegen.md`](../../docs/reference/typegen.md) for the full workflow.
